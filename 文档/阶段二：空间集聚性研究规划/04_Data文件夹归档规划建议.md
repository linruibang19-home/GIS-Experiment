# Data 文件夹归档规划建议

## 1. 当前 Data 目录理解

当前 `Data/` 目录混合了四类内容：

1. 原始高企数据和历史过程归档；
2. 阶段一完整底层数据副本；
3. 外部影响因素数据，如 POI、人口、行政区；
4. 早期处理脚本、输出、中间文件和可视化。

这说明 Data 目录现在更像“总数据仓库”，不适合直接作为阶段二、三、四的输入目录。

## 2. 后续归档原则

建议先不移动现有 Data 文件，先建立逻辑规则：

1. `阶段一：企业动态变量研究/` 作为正式底板主来源；
2. `Data/高新技术企业数据库/_历史过程与归档/` 只作追溯，不作为实验输入；
3. `Data/高新技术企业数据库/第一阶段：完整底层数据/` 可作为阶段一成果镜像，但后续优先用 `阶段一` 目录；
4. `Data/POI数据/`、`Data/人口数据/`、`Data/行政区数据/` 是阶段三影响因素研究的原始来源；
5. 每个正式实验阶段应复制输入到本阶段目录，而不是直接改 Data 原文件。

## 5. 不建议直接作为正式输入的内容

1. `_历史过程与归档/` 内原始和早期文件；
2. `数据处理/output/中间数据/`；
3. 旧 clip 格网 ID 体系的中间表；
4. ArcGIS GDB 内部文件；
5. 重复副本，除非用于核验来源。

## 6. 下一步建议

先制作一份正式数据清单，字段包括：

```text
数据名称、路径、年份、空间尺度、行数、字段数、用途、是否正式输入、备注
```

确认清单后，再决定是否实际移动或复制 Data 文件。
