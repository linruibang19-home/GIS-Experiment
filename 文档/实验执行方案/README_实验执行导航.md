# 广州市高新技术企业空间集聚研究：实验执行导航

## 1. 文档定位

本目录用于指导论文后续实验的实际执行。原有概念性方案继续保留在：

```text
文档/实验方案/
├─ gz_hitech_gis_experiment_framework.md
└─ gz_hitech_spatiotemporal_prediction_redesign.md
```

本目录不替代原有方案，而是将其拆解为可逐步执行、可质检、可复现的操作手册。

论文题目建议统一为：

> 广州市高新技术企业空间集聚的多尺度影响因素与时空预测研究

## 2. 实验主线

```text
阶段一：数据底板质检与冻结
        ↓
阶段二：空间集聚性研究
        ↓
阶段三：多尺度影响因素研究
        ↓
阶段四：年度时空预测研究
        ↓
阶段五：成果汇总、论文图表与复现归档
```

三个论文核心内容分别对应：

| 论文内容 | 执行阶段 | 核心问题 |
|---|---|---|
| 空间集聚性研究 | 阶段二 | 企业是否集聚、在哪里集聚、热点如何演变 |
| 空间分布影响因素研究 | 阶段三 | 哪些因素影响集聚，作用尺度和空间异质性如何 |
| 时空预测性研究 | 阶段四 | 下一年企业存量、进入、退出、续认和热点如何变化 |

阶段一和阶段五是保证论文质量的支撑阶段。

## 6. 执行原则

1. `阶段一：企业动态变量研究` 作为数据归档目录，后续只读使用。
2. 后续分析统一使用主格网标准 ID：`grid_id = RxxxCxxx`。
3. 有效格网分析统一使用 `valid_mask=1`。
4. 深度学习或完整矩阵任务使用完整格网面板，并保留 `valid_mask=0` 的市外补齐格网。
5. 边界格网默认保留，并使用剔除 `boundary_mask=1` 的结果作为稳健性检验。
6. 任何统计结果必须附带输入文件、参数、软件版本、运行时间和输出路径。
7. ArcGIS Pro Python 窗口中的代码应同步保存为独立 `.py` 文件，避免只保留在 `.pyHistory` 中。

## 7. 推荐执行顺序

| 顺序 | 任务 | 是否可以立即开始 |
|---:|---|---|
| 1 | 阶段一质检、迁移变量派生、正式底板冻结 | 可以 |
| 2 | 年度分布图、Global Moran's I、LISA、Gi* | 可以 |
| 3 | 时空立方体与热点演化类型识别 | 可以 |
| 4 | 外部影响因素数据搜集与格网化 | 部分可以 |
| 5 | GeoDetector、计数回归、MGWR | 需完成协变量库 |
| 6 | 预测专用面板和滚动验证 | 需完成阶段一补充变量 |
| 7 | 2025 年预测和论文图表整理 | 需完成模型比较 |

## 8. 当前必须关注的问题

1. 企业存在跨格网迁移：22,657 家企业中有 777 家出现多个主格网位置，至少发生 780 次格网变化。
2. 格网层面的存量平衡式需要加入迁入和迁出项。
3. 2015 年是面板起始年，不能将全部进入量解释为真实新增。
4. 2024 年缺少下一年状态，`exit_next_cnt=0` 属于右删失，不能直接作为真实退出零值训练模型。
5. 企业点位表中存在少量广州市外区县文本标签，需要以空间连接结果生成标准区县字段。
6. 影响因素数据仍不完整，目前主要已有 2022 年 POI、2024 年人口栅格和行政区图层。
