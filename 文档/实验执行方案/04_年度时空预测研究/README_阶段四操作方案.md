# 阶段四：年度时空预测研究

## 1. 研究目标

本阶段在已经完成空间集聚实验和多尺度影响因素实验的基础上，预测下一年度 500m 主格网尺度下的高新技术企业空间变化。

核心预测对象包括：

1. 活跃高新技术企业存量；
2. 新进入高新资格体系的企业数量；
3. 高新资格退出风险；
4. 续认能力；
5. 增长潜力区；
6. 热点延续、新兴热点和衰退风险区。

本阶段的重点不是单纯追求一个黑箱预测结果，而是将“空间集聚格局”“影响因素机制”和“年度预测”连接起来，形成可以写入论文第六章的时空预测研究。

## 2. 方法定位

### 2.1 不建议继续将 ConvLSTM 作为主模型

开题报告中原计划使用 ConvLSTM。结合当前真实数据底板，该方法不适合作为论文主模型。

主要原因：

1. 2015 至 2024 年只有 10 个真实年度时间片；
2. 月度或季度拆分会制造伪时序；
3. 企业高新资格具有明确的 3 年有效期和续认制度；
4. 格网计数数据零值比例高，且具有明显过度离散；
5. 硕士论文需要兼顾预测精度、机制解释和政策含义。

ConvLSTM可以作为探索性对照，但不应作为核心结论依据。

### 2.2 推荐主模型：事件流时空图神经网络

为了给论文增加合理的深度学习创新，推荐将原有 EF-STP 事件流状态转移模型升级为：

```text
EF-STGNN：基于事件流的时空图神经网络预测模型
Event-flow Spatio-Temporal Graph Neural Network
```

该方法的核心思想是：

```text
不是直接预测下一年企业总量，
而是分别预测进入、退出、续认和迁移等事件流，
再用事件流平衡关系更新下一年活跃存量。
```

同时，使用图神经网络表达 500m 格网之间的空间邻接、产业相似性和影响因素相似性。这样既保留深度学习创新，又不会违背年度数据的真实时间粒度。

## 3. 为什么 STGNN 比 ConvLSTM 更适合本研究

| 维度 | ConvLSTM | EF-STGNN |
|---|---|---|
| 空间表达 | 依赖完整规则矩阵 | 使用有效格网图结构 |
| 时间要求 | 更适合高频长序列 | 可用于短年度面板 |
| 空间关系 | 默认局部卷积邻域 | 可融合邻接、距离、产业相似、影响因素相似 |
| 数据利用 | 需要完整矩阵输入 | 可直接使用 29,755 个有效主格网 |
| 机制解释 | 较弱 | 可按进入、退出、续认解释 |
| 与前文衔接 | 与影响因素章节衔接弱 | 可直接接入 MGWR/GeoDetector 识别出的主导因素 |
| 论文创新 | 常规深度学习套用 | 事件流 + 图神经网络 + 多尺度因素融合 |

因此，本研究建议将深度学习创新点定位为：

> 基于空间邻接、影响因素相似性和企业资格事件流的年度时空图神经网络预测。

### 3.1 可行性判断

本方案具有可行性，但必须控制模型复杂度，不能按照交通流、气象等高频数据场景直接套用大模型。

可行的原因：

1. 空间样本充足。当前有效主格网为 29,755 个，完整矩形格网为 68,930 个，虽然年度时间片只有 10 年，但空间节点数量足够支撑图结构建模。
2. 空间关系明确。500m 主格网具有稳定的 `grid_id`、`row_id`、`col_id`、质心坐标和邻接关系，可以自然构建空间图。
3. 企业动态变量清晰。阶段一已经构建 `active_cnt`、`entry_cnt`、`persist_cnt`、`renew_cnt`、`exit_next_cnt` 等事件变量，适合多任务预测。
4. 影响因素可接入。阶段三的创新资源、交通、政策、产业基础等协变量可以作为节点属性，也可以用于构建影响因素相似图。
5. 与论文主线一致。EF-STGNN不是孤立使用深度学习，而是承接空间集聚实验和影响因素实验，预测结果也可以解释为进入、退出、续认和增长潜力。

需要控制的风险：

1. 年份窗口少，不能训练过深模型；
2. 不应使用季度或月度伪数据扩充；
3. 不能只报告深度学习精度，必须与 Persistence、XGBoost、统计型 EF-STP 等模型对比；
4. 如果深度学习模型精度不是最优，也可以将其定位为“空间依赖和事件流机制融合”的创新模型，而不是唯一最优模型。

### 3.2 推荐该方案的依据

推荐 EF-STGNN 不是简单为了使用深度学习，而是基于以下方法依据和数据适配性判断：

1. STGCN 证明图卷积可以用于同时刻画空间网络依赖和时间变化，适合将规则或非规则空间单元表达为图结构。
2. DCRNN 证明图上的扩散过程和循环网络可以结合，用于建模空间传播关系和时间依赖。
3. Graph WaveNet 进一步提出自适应邻接矩阵，可学习数据中未被显式空间邻接捕捉到的隐藏依赖。
4. 本研究的 500m 格网不是交通传感器网络，但同样具有“节点-边-时间序列”的结构：格网是节点，邻接/距离/相似性是边，2015-2024 年企业动态变量是年度时序。
5. 与这些方法不同，本研究额外引入高新技术企业资格事件流，将预测对象从单一企业数量拆分为进入、退出、续认和迁移，使模型更符合研究对象的制度机制。

因此，本研究采用的是“借鉴时空图神经网络思想，并结合高新技术企业资格生命周期进行改造”的方案，而不是直接照搬交通预测模型。

## 6. 事件流平衡关系

为避免公式在部分 Markdown 查看器中显示乱码，本方案使用纯文本公式。论文正式写作时可以再转换为 LaTeX。

### 6.1 简化事件流公式

```text
A_hat(g, t+1) = A(g, t) + I_hat(g, t+1) - O_hat(g, t+1)
```

含义：

| 符号 | 说明 |
|---|---|
| `A(g,t)` | 格网 g 在年份 t 的活跃企业存量 |
| `I_hat(g,t+1)` | 预测的下一年进入代理量 |
| `O_hat(g,t+1)` | 预测的下一年退出代理量 |
| `A_hat(g,t+1)` | 预测的下一年活跃企业存量 |

### 6.2 加入企业迁移后的正式公式

由于部分企业存在跨格网迁移，正式模型建议使用：

```text
A_hat(g, t+1)
  = A(g, t)
  + I_hat(g, t+1)
  - O_hat(g, t+1)
  + M_in_hat(g, t+1)
  - M_out_hat(g, t+1)
```

其中：

| 符号 | 说明 |
|---|---|
| `M_in_hat(g,t+1)` | 预测的下一年迁入数量 |
| `M_out_hat(g,t+1)` | 预测的下一年迁出数量 |

续认量 `R(g,t+1)` 不直接重复加到活跃存量中，而是作为资格维持机制的重要预测目标，也可以作为退出风险模型的解释变量。

### 6.3 增长潜力指标

```text
GrowthPotential(g, t+1)
  = I_hat(g, t+1)
  - O_hat(g, t+1)
  + M_in_hat(g, t+1)
  - M_out_hat(g, t+1)
```

解释：

```text
GrowthPotential > 0：预测为增长潜力格网
GrowthPotential = 0：预测为稳定格网
GrowthPotential < 0：预测为收缩或风险格网
```

#### 7.4.1 注册资本变量

注册资本可以表示格网内企业的资本强度和企业发展基础。

建议构建：

| 字段 | 含义 |
|---|---|
| `capital_mean` | 格网年度企业注册资本均值 |
| `capital_median` | 格网年度企业注册资本中位数 |
| `capital_sum` | 格网年度企业注册资本总量 |
| `capital_log_mean` | 注册资本均值取对数 |
| `capital_high_ratio` | 高注册资本企业占比 |
| `capital_weighted_active` | 注册资本加权活跃企业存量 |

研究用途：

1. 空间集聚研究中，可比较“企业数量集聚”和“资本强度集聚”是否一致；
2. 影响因素研究中，可作为企业结构控制变量，避免将资本较强格网的集聚误判为单纯区位因素作用；
3. 预测研究中，可用于预测续认能力和退出风险，因为资本基础较强的企业通常更可能具备持续研发投入能力；
4. 政策解释中，可识别“数量多但资本弱”的培育型区域和“数量少但资本强”的高质量集聚区域。

注意：

```text
注册资本受天眼查查询时点影响，论文中应将其解释为企业资本结构属性，不宜过度解释为每一历史年份的真实资本变化。
```

#### 7.4.2 企业规模变量

企业规模可以反映格网内企业结构稳定性和抗风险能力。

建议构建：

| 字段 | 含义 |
|---|---|
| `n_size_微型` | 微型企业数量 |
| `n_size_小型` | 小型企业数量 |
| `n_size_中型` | 中型企业数量 |
| `n_size_大型` | 大型企业数量 |
| `small_micro_ratio` | 小微企业占比 |
| `medium_large_ratio` | 中大型企业占比 |
| `size_entropy` | 企业规模多样性 |

研究用途：

1. 空间集聚研究中，可分析小微企业和中大型企业集聚热点是否不同；
2. 影响因素研究中，可检验政策平台、交通可达性、创新资源对不同规模企业的作用差异；
3. 预测研究中，小微企业占比较高的格网可作为退出风险和续认风险的重要特征；
4. 续认模型中，中大型企业占比可作为资格维持能力的代理变量；
5. 政策建议中，可区分“孵化培育型格网”和“成熟企业集聚型格网”。

#### 7.4.3 高新技术企业门类变量

高新技术企业门类可以反映格网的技术结构和产业专业化方向。

建议构建：

| 字段 | 含义 |
|---|---|
| `n_cat_电子信息` | 电子信息企业数量 |
| `n_cat_高技术服务` | 高技术服务企业数量 |
| `n_cat_先进制造与自动化技术` | 先进制造与自动化企业数量 |
| `n_cat_新材料技术` | 新材料企业数量 |
| `n_cat_生物与新医药技术` | 生物医药企业数量 |
| `category_entropy` | 门类多样性指数 |
| `dominant_category` | 格网主导门类 |
| `category_specialization_index` | 门类专业化指数 |

研究用途：

1. 空间集聚研究中，可识别不同技术门类的热点差异；
2. 影响因素研究中，可比较电子信息、生物医药、先进制造等门类对交通、政策、创新资源的敏感性差异；
3. 预测研究中，门类结构可作为节点特征，帮助模型识别不同技术产业的增长路径；
4. 图神经网络中，可用门类结构相似性构建 `W_factor` 或 `W_history`，表达“非邻接但产业结构相似”的格网关系；
5. 政策建议中，可识别不同技术方向的重点培育区域。

门类多样性可以使用 Shannon entropy：

```text
category_entropy(g,t)
  = - sum over c of p(c,g,t) * log(p(c,g,t))
```

其中：

```text
p(c,g,t) = 某门类 c 在格网 g、年份 t 中的企业占比
```

## 8. 深度学习主方案：EF-STGNN

### 8.1 模型定位

EF-STGNN 是本阶段最推荐的深度学习创新方案。

其作用是：

```text
用图神经网络学习格网之间的空间依赖，
用时间模块学习年度变化趋势，
用多任务输出预测进入、退出、续认、迁移和热点，
最后用事件流平衡关系更新活跃存量。
```

### 8.2 图节点

每个有效主格网为一个节点：

```text
node = grid_id
N = 29,755 个广州市内有效主格网
```

如果深度学习训练内存不足，可以采用：

1. 仅保留有企业或邻近有企业的格网；
2. 按行政区或空间块训练子图；
3. 采用 GraphSAGE / mini-batch 采样；
4. 使用完整图预测、子图训练。

### 8.3 图边设计

建议构建多关系图，而不是只用简单 Queen 邻接。

| 图关系 | 说明 | 用途 |
|---|---|---|
| `W_adj` | Queen 或 Rook 邻接 | 表达500m格网空间接壤 |
| `W_dist` | KNN 或距离衰减 | 表达近距离空间影响 |
| `W_factor` | 影响因素相似性 | 表达创新资源、交通、政策等环境相似 |
| `W_history` | 历史企业变化相似性 | 表达企业增长轨迹相似 |
| `W_adaptive` | 模型自学习邻接 | 捕捉隐藏空间关联 |

推荐主方案：

```text
W = W_adj + W_dist + W_factor + W_adaptive
```

如果论文写作需要更简洁，可以写成：

```text
本文构建空间邻接图、距离邻近图和影响因素相似图，并引入可学习自适应邻接矩阵，以共同刻画格网之间的显性与隐性空间联系。
```

企业注册资本、企业规模和高新技术企业门类也可以参与图边构建。

建议：

| 属性 | 图边构建方式 | 含义 |
|---|---|---|
| 注册资本结构 | 资本均值、资本总量、资本高值占比的相似性 | 识别资本结构相似的格网 |
| 企业规模结构 | 小微占比、中大型占比、规模熵的相似性 | 识别企业发展阶段相似的格网 |
| 高新技术门类结构 | 八大门类占比向量的余弦相似性 | 识别产业技术结构相似的格网 |

可构建：

```text
W_enterprise_structure
  = similarity(
      capital_features,
      size_features,
      category_features
    )
```

最终多关系图可写为：

```text
W_total
  = W_adj
  + W_dist
  + W_factor
  + W_enterprise_structure
  + W_adaptive
```

解释：

```text
W_adj 表示地理邻近；
W_factor 表示外部区位条件相似；
W_enterprise_structure 表示企业结构相似；
W_adaptive 表示模型学习到的隐性联系。
```

### 8.4 输入张量

每个训练样本使用过去 `k` 年信息预测下一年。

推荐：

```text
k = 3
输入：X(t-2), X(t-1), X(t)
输出：Y(t+1)
```

输入格式：

```text
X_input shape = [num_nodes, time_steps, num_features]
```

其中：

```text
num_nodes = 有效格网数量
time_steps = 3
num_features = 企业动态变量 + 企业属性结构变量 + 影响因素变量 + 集聚结果变量
```

企业属性结构变量建议至少包括：

```text
capital_log_mean
capital_high_ratio
small_micro_ratio
medium_large_ratio
category_entropy
dominant_category_onehot
主要门类占比变量
```

### 8.5 模型结构

建议采用轻量结构，避免时间片过少导致过拟合。

```text
输入层
  ↓
时间模块：GRU 或 Temporal Convolution
  ↓
空间模块：Graph Convolution / Graph Attention
  ↓
多任务输出层
  ├─ entry_head：预测进入量
  ├─ exit_head：预测退出量
  ├─ renew_head：预测续认量
  ├─ move_in_head：预测迁入量
  ├─ move_out_head：预测迁出量
  └─ hotspot_head：预测热点概率
  ↓
事件流平衡层
  ↓
active_hat_next
```

### 8.6 多任务输出

EF-STGNN 不建议只预测 `active_next`。更合理的是多任务预测：

| 输出头 | 目标变量 | 建议损失函数 |
|---|---|---|
| `entry_head` | `target_entry_next` | Poisson Loss / Negative Binomial Loss |
| `exit_head` | `target_exit_next` | Poisson Loss / Negative Binomial Loss |
| `renew_head` | `target_renew_next` | Poisson Loss / Binomial Loss |
| `move_in_head` | `target_move_in_next` | Poisson Loss |
| `move_out_head` | `target_move_out_next` | Poisson Loss |
| `hotspot_head` | `target_hotspot_next` | Binary Cross Entropy / Focal Loss |
| `active_event_flow` | `target_active_next` | MAE / Huber Loss |

总损失：

```text
Loss_total
  = w1 * Loss_entry
  + w2 * Loss_exit
  + w3 * Loss_renew
  + w4 * Loss_move
  + w5 * Loss_hotspot
  + w6 * Loss_active
```

权重可以先设置为：

```text
w1 = w2 = w3 = w4 = 1
w5 = 0.5
w6 = 1
```

后续根据验证集表现调整。

### 8.7 事件流平衡层

深度学习模型输出事件流后，再计算活跃存量：

```text
active_hat_next
  = active_t
  + entry_hat_next
  - exit_hat_next
  + move_in_hat_next
  - move_out_hat_next
```

约束：

```text
active_hat_next = max(active_hat_next, 0)
```

这样做的优点是：

1. 预测结果符合企业资格事件逻辑；
2. 可以分别解释进入、退出和续认机制；
3. 深度学习结果不是完全黑箱；
4. 与阶段一动态变量构建高度一致。

### 8.8 训练样本构造

由于只有 10 年，不做随机年份拆分。

滚动样本如下：

| 输入年份 | 预测年份 |
|---|---:|
| 2015, 2016, 2017 | 2018 |
| 2016, 2017, 2018 | 2019 |
| 2017, 2018, 2019 | 2020 |
| 2018, 2019, 2020 | 2021 |
| 2019, 2020, 2021 | 2022 |
| 2020, 2021, 2022 | 2023 |
| 2021, 2022, 2023 | 2024 |

注意：

```text
年份窗口少是客观限制，因此模型必须轻量化，并与统计模型和机器学习模型对照。
```

### 8.9 训练策略

建议采用：

1. 空间块 mini-batch；
2. 早停法；
3. Dropout；
4. L2 正则化；
5. 只使用 1 至 2 层图卷积；
6. 不使用过深 Transformer；
7. 不做季度或月度伪扩展。

## 9. 可选深度学习对照方案

### 9.1 STGCN 对照

STGCN 使用图卷积捕捉空间依赖，用时间卷积捕捉时间变化。它比 ConvLSTM 更适合图结构格网。

在本研究中，STGCN 可以作为 EF-STGNN 的简化对照：

```text
输入：过去3年格网节点特征
图：Queen邻接或KNN邻接
输出：下一年 active_cnt
```

局限：

```text
只直接预测活跃存量，事件机制解释弱于 EF-STGNN。
```

### 9.2 Graph WaveNet 对照

Graph WaveNet 的优势是可以学习自适应邻接矩阵，适合发现不完全由地理邻接决定的隐性空间联系。

在本研究中，它可以用于：

1. 检查是否存在跨区隐性关联；
2. 发现非邻接但变化模式相似的格网；
3. 作为深度学习预测对照。

局限：

```text
模型复杂度高于 STGCN，年度时间片较少时需要严格控制参数量。
```

### 9.3 TFT 不建议作为主模型

Temporal Fusion Transformer 具有较强的多步预测和变量解释能力，但本研究年度时间片较少，且重点是空间格网关系。TFT可以作为企业或格网时间序列的补充模型，但不建议作为主模型。

### 9.4 ConvLSTM 探索性对照

ConvLSTM只建议保留为探索性对照。

输入：

```text
完整规则格网：305 × 226
过去3年矩阵 → 下一年矩阵
valid_mask 区分市内有效格网与市外补齐格网
```

要求：

1. 不使用伪季度数据；
2. 不作为主结论；
3. 不宣称其最优；
4. 明确说明时间窗口不足；
5. 若效果不好，也可作为“高频时空深度模型不适合该年度数据条件”的方法讨论。

## 10. 操作步骤

### 步骤 1：建立基准模型

必须先完成简单基准。

| 编号 | 模型 | 预测方式 |
|---|---|---|
| B0 | Zero Baseline | 全部预测为 0 |
| B1 | Persistence | `A(t+1) = A(t)` |
| B2 | Moving Average | 最近三年均值 |
| B3 | Linear Trend | 最近三年线性趋势 |

输出：

```text
2、基准模型/baseline_metrics.csv
```

### 步骤 2：统计型事件流子模型

#### 2.1 进入模型

目标：

```text
target_entry_next
```

候选模型：

- 负二项；
- ZINB；
- hurdle；
- XGBoost Poisson；
- 两阶段模型：是否进入 + 正值数量。

#### 2.2 退出模型

目标：

```text
target_exit_next
```

候选模型：

- 企业级二项 Logit；
- 格网级 ZINB；
- XGBoost；
- 生存分析作为补充。

注意：

```text
2024 年退出量不能参与训练。
```

#### 2.3 续认模型

建议优先采用企业级模型：

```text
R(i, t+1) follows Bernoulli(p(i, t+1))
```

将企业级预测概率聚合到格网：

```text
R_hat(g, t+1) = sum of p_hat(i, t+1) for firms located in grid g
```

需要派生：

```text
expiry_candidate_next
```

表示下一年面临到期或续认判断的企业。

#### 2.4 迁移模型

当前迁移数量相对企业总量较小，可采用两种处理。

| 方案 | 说明 |
|---|---|
| 主方案 | 使用历史平均迁移率和空间邻域特征预测迁入、迁出 |
| 简化方案 | 在 EF-STP 或 EF-STGNN 中作为修正项，在稳健性分析中讨论 |

如果迁移预测效果较差，应明确其为模型误差来源。

### 步骤 3：机器学习直接预测对照

使用：

- Random Forest；
- XGBoost；
- LightGBM，可选。

任务：

```text
直接预测 target_active_next
```

必须报告：

- MAE；
- RMSE；
- Spearman；
- Top-K Hit Rate；
- 热点 Precision、Recall、F1；
- 残差 Moran's I；
- SHAP 特征重要性。

### 步骤 4：EF-STGNN 深度学习模型

按照第 8 节执行。

模型结果必须与以下模型比较：

1. Persistence；
2. Moving Average；
3. XGBoost；
4. 统计型 EF-STP；
5. STGCN 或 Graph WaveNet 对照。

如果 EF-STGNN 在数值误差上不是最优，但在热点识别、增长潜力区识别或事件流解释上表现更好，也可以作为合理贡献。

### 步骤 5：空间 Markov 与 LISA-Markov

该部分用于预测空间等级和热点状态，不追求精确企业数量。

状态划分：

| 状态 | 含义 |
|---|---|
| S0 | 无企业 |
| S1 | 非零低值 |
| S2 | 非零中低值 |
| S3 | 非零中高值 |
| S4 | 非零高值 |

输出：

```text
6、空间Markov/markov_transition_matrix.csv
6、空间Markov/lisa_markov_transition_matrix.csv
6、空间Markov/potential_upgrade_grid.csv
6、空间Markov/persistent_hotspot_grid.csv
```

### 步骤 6：贝叶斯时空模型

作为增强模型，用于输出：

- 预测均值；
- 95% 可信区间；
- 增长概率；
- 热点概率；
- 退出风险概率；
- 空间不确定性。

推荐：

```text
R CARBayesST 或 R-INLA
```

如果时间和环境有限，可以将该部分作为加分项，不阻塞主实验完成。

## 13. 未来年度预测

1. 必须优于 Persistence；
2. 滚动验证结果稳定；

4. 残差空间自相关较弱；

6. 深度学习模型不能只报告精度，必须报告事件流解释结果。

2025 年

## 15. 论文中可写的创新表述

可以在论文中写为：

```text
考虑到高新技术企业数据的年度时间粒度和资格认定周期特征，本文未直接采用面向高频栅格序列的 ConvLSTM 作为主模型，而是构建基于事件流的时空图神经网络模型。该模型以 500m 格网为图节点，综合空间邻接关系、距离邻近关系、影响因素相似性和可学习自适应邻接关系，刻画格网之间的显性与隐性空间联系；同时，将下一年度活跃企业存量拆解为进入、退出、续认和迁移等事件过程，通过多任务学习预测各类事件流，并利用存量-流量平衡关系更新下一年度企业活跃存量。该方法既保留了深度学习对复杂时空非线性关系的表达能力，又符合高新技术企业资格生命周期的制度逻辑。
```
