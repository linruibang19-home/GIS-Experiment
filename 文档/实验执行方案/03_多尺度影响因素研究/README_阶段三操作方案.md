# 阶段三：多尺度影响因素研究

## 1. 研究目标

回答以下问题：

1. 哪些因素影响高新技术企业空间集聚？
2. 影响因素对活跃、进入、续认和退出是否不同？
3. 因素之间是否存在交互增强？
4. 不同因素的作用尺度是否不同？
5. 不同区域是否具有空间异质性？

## 2. 多尺度定义

论文中的“多尺度”必须明确，不宜只依赖 MGWR 带宽解释。

建议采用三层尺度设计：

| 层次 | 设计 | 作用 |
|---|---|---|
| 基础空间单元 | 500m 主格网 | 与企业数据底板一致 |
| 邻域变量尺度 | 500m、1km、2km、3km 缓冲区 | 比较影响范围 |
| 稳健性分析尺度 | 1km、2km 聚合格网 | 检查 MAUP 敏感性 |

MGWR 用于补充识别不同解释变量的局部作用尺度。

## 7. 操作步骤

### 步骤 1：统一空间参考和时间口径

统一投影：

```text
EPSG:4547
CGCS2000 / 3-degree Gauss-Kruger CM 114E
```

每项数据必须记录：

- 数据来源；
- 数据年份；
- 坐标系；
- 原始字段；
- 清洗规则；
- 格网化方法；
- 是否为静态变量；
- 是否存在时间错配。

输出：

```text
10、质检报告/covariate_source_catalog.csv
10、质检报告/covariate_dictionary.md
```

### 步骤 2：POI 分类

当前 POI 数据包含大类和中类字段。先构建分类映射表，再进行格网统计。

建议至少分类：

```text

输出：

```

### 步骤 3：生成多尺度邻域变量

对点、线、面、栅格分别使用：

| 数据类型 | 方法 |
|---|---|
| 点 | 格网内数量、缓冲区数量、核密度、最近距离 |
| 线 | 缓冲区内总长度、道路密度 |
| 面 | 叠加面积、覆盖率、政策区 dummy |
| 栅格 | 分区统计均值、总量、标准差 |

缓冲距离：

```text
500m、1km、2km、3km
```

变量命名示例：

```text

poi_industrial_cnt_b1000
poi_industrial_cnt_b2000
poi_industrial_cnt_b3000
```

### 步骤 5：描述统计和共线性

检查：

1. 缺失率；
2. 零值比例；
3. 极端值；
4. 偏态；
5. Pearson 和 Spearman 相关；
6. VIF；
7. 多尺度变量之间的冗余。

处理建议：

- 距离变量可使用 `log1p`；
- 密度变量可使用 `log1p`；
- 极端值可 Winsorize；
- 高相关多尺度变量不要同时进入同一个模型；
- 标准化变量用于 MGWR 和机器学习。

### 步骤 6：GeoDetector

作用：探索空间分异解释力和交互增强。

建议：

1. 以 2024 年横截面为主；
2. 增加 2018、2021 年作为对比；
3. 连续变量尝试分位数、自然断点和 K-means 离散化；
4. 每个变量尝试 4 至 7 类；
5. 保留分类稳健的变量；
6. 对排名靠前的变量执行交互探测。

因变量：

```text
active_cnt
entry_cnt
renew_cnt
exit_next_cnt
```

输出：

```text
5、GeoDetector/factor_detector_YYYY.csv
5、GeoDetector/interaction_detector_YYYY.csv
9、论文图件/fig_geodetector_q_rank.png
9、论文图件/fig_geodetector_interaction_heatmap.png
```

### 步骤 7：计数回归

现有面板零值比例较高：

| 变量 | 零值比例 |
|---|---:|
| `active_cnt` | 90.97% |
| `entry_cnt` | 95.74% |
| `exit_next_cnt` | 97.92% |
| `renew_cnt` | 97.40% |

主模型建议：

| 因变量 | 主模型 |
|---|---|
| `active_cnt` | 负二项、ZINB |
| `entry_cnt` | hurdle 或 ZINB |
| `renew_cnt` | 二项模型或 ZINB |
| `exit_next_cnt` | 二项风险模型或 ZINB |

模型顺序：

```text
M0：仅位置与年份控制变量
M1：加入路径依赖
M2：加入空间滞后
M3：加入创新资源和产业基础
M4：加入交通和城市功能
M5：加入政策平台
```

控制变量：

- 年份固定效应；
- 区县固定效应；
- `cover_ratio`；
- 边界格网标记；
- 企业规模结构；
- 技术门类结构。

### 步骤 8：MGWR

MGWR 用于补充回答：

```text
不同因素是否在不同空间尺度上起作用？
局部作用方向和强度是否存在空间差异？
```

建议：

1. 使用 `log1p(active_cnt)` 作为主因变量；
2. 从回归和 GeoDetector 中选择 4 至 6 个核心解释变量；
3. 对解释变量标准化；
4. 检查局部共线性；
5. 输出每个变量的带宽、局部系数和局部显著性；
6. 不将 MGWR 作为因果识别工具。

输出：

```text
7、MGWR/mgwr_summary.txt
7、MGWR/mgwr_local_coefficients.gpkg
9、论文图件/map_mgwr_coefficient_*.png
```

### 步骤 9：稳健性检验

至少完成：

1. 500m、1km、2km 聚合格网；
2. 500m、1km、2km、3km 邻域变量；
3. 保留与剔除边界格网；
4. 活跃存量与注册资本加权存量；
5. Queen 邻域与 KNN 邻域；
6. 剔除极端高值格网；
7. 静态变量模型与动态变量模型分开报告。

## 8. 解释边界

1. 单一年份 POI 和人口数据只能支持空间关联解释，不宜直接表述为动态因果关系。
2. 政策平台变量如果缺少设立年份，只能作为空间背景变量。
3. 路径依赖变量和邻域滞后变量必须使用前一期值。
4. MGWR 反映空间异质性，不等同于因果识别。
5. GeoDetector 用于探索性筛选和交互分析，不应替代回归模型。
